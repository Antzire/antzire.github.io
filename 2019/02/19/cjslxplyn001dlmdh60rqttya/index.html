Github+Hexo+Nodejs+Qiniu 搭建个人博客

发表于 2019-02-19 | 分类于 Github

跟很多同学一样，想搭建自己的Blog，一开始尝试了阿里云的虚拟主机加WordPress，部署好了之后发现WordPress中的Markdown只能以插件的方式存在，个人觉得不太好用，而且代码高亮也有很多问题，然后阿里云的虚拟机主机莫名奇妙的出现了一次无法访问，开工单咨询说是可能是遭到了攻击，让我购买阿里云的防护功能组件，我听了呵呵一笑。
后来就在网上看还有没有其他方式可以搭建自己相对稳定的Blog，看到了各种大神的关于Github Pages 搭建Blog的方法后决定尝试下，网上的最多的方法就是Github+Jeklly和Github+Hexo的方式，仔细浏览后发现很多帖子说Jeklly生成文章的方式较慢，最终决定选择Github+Hexo的方式来搭建自己的Blog。

1. 资源描述

  • Github 博客主平台，官方建议使用空间保持在1GB以内
  • Godaddy 注册自己的域名，可选国内服务商需要备案
  • Node.js 基于Chrome V8 JavaScript引擎的JavaScript框架
  • Hexo 基于Node.js的静态博客框架
  • NexT Hexo的主题模板，可选其他，参考Hexo官网主题
  • Qiniu 免费10G图片云存储，作为博客图床

2. 创建Github Pages

  • 注册Github
    打开Github官网链接,输入Username，Email，Password
  • 创建Github Pages
    点击Start a project创建新的仓库，例如我注册用户名为cnethan，我的仓库名就是cnethan.github.io
  • 选择一个Github Pages的模板
    点击Settings，找到Github Pages选择Change theme
  • 选择Slate模板进行测试，个人习惯，可以选择其他的
  • 选择好模板后点击底部的Commit changes
  • 新开一个网页输入cnethan.github.io看到Welcome to Github Pages,那么恭喜你一个简单基于Github的博客已经搭建成功了，不要着急，下面我们会继续改造它

3. Github配置

  • 安装git，参考Pro Git
    注意 : 我的操作系统为Fedora 29，Windows系统需要在安装好git后，使用cmd或者右键菜单的Git bash来查看版本和创建，如果你使用查看版本的命令没有问题，就可以使用后续的创建命令，如果有问题请邮箱联系我

  • 安装完成后查看git版本

  • 配置全局用户名和邮箱

  • 生成SSH key

  • 查看SSH key

  • 将SSH key添加到Github
    点击你Github头像的小三角，在下拉列表中找到Settings，选中SSH and GPG keys，点击New SSH key，将你查看到的SSH key 复制并粘贴

  • 测试是否添加成功

4. 域名配置

  • 注册域名
    打开Godaddy官网，注册并选择域名，支持支付宝支付
  • 查看购买的域名，点击Domains下的All Domains标签
  • 配置解析
    点击所购买的域名后拉到底部点击Manage DNS
  • 添加记录
    添加指向Github Pages的A记录，IP地址如下：
    185.199.108.153
    185.199.109.153
    185.199.110.153
    185.199.111.153

  • 查看解析是否生效，约15分钟左右

  • 开启自定义域名https访问
    自定义域名后默认的链接为http，为了安全起见我们开启Github Pages的https

  • Github添加CNAME，可开始自定义域名的https访问
    在cnethan.github.io的仓库下新建一个名称为大写CNAME的文件，内容为你的域名，也可以是二级域名

8. 七牛图床

  • 七牛官网
